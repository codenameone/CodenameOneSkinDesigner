Skin Designer Help

The skin designer tool allows you to visually design and save a Codename One skin file that you can use to simulate a custom device type. To use this tool you need two pictures of the device in portrait and landscape mode. The screen area of these pictures must match the device screen size!

TIP: To find files like that just search for the name of the device you are looking for followed by "mockup". This will bring up device mockups you can often use. FYI be sure to read the license of the graphic you use if you intend to publish this, many creators require attribution!

Creating a Skin

You will need to select the device image in portrait/landscape, you can then enter the screen dimensions in pixels that should match the area within the graphic.
You can enter the X/Y position or use the hand "pan tool" to visually position the screen within the graphic.

This process should be repeated both for portrait and landscape images respectively.

You then need to enter a set of values to indicate details about the skin as described below:

  • Native Theme - the underlying native theme of the device, e.g. iOS devices use an iOS 7+ native theme
  • Platform name - This affects the Display platform name
  • Tablet - this indicates if the isTablet() method will return true or false
  • System Font Family - the default builtin font family. Notice that this only applies to system fonts and not to native fonts
  • Proportional Font Family - the proportional builtin font family. Notice that this only applies to system fonts and not to native fonts
  • Monospace Font Family - the monospace font family. Notice that this only applies to system fonts and not to native fonts
  • Small font size - when you pick "small" for the font size this is the size it will have in pixels
  • Medium font size - when you pick "medium" for the font size this is the size it will have in pixels
  • Large font size - when you pick "large" for the font size this is the size it will have in pixels
  • Pixel Ratio - the number of pixels per millimeter on the device. To convert a PPI (pixels per inch) value to millimeters just divide the value by 25.4.
    E.g. a device with 264 ppi will have 264 / 25.4 = 10.3937008 pixels per millimeter
  • Platform overrides - These represent the theme override behavior which you can use to override platform specific behavior on the device

Saving & Testing

You can save the skin by pressing the save button on the top right. In the desktop version it will reside in the .SkinDesigner directory in your home folder. In the web version you will be prompted to save the skin file.

Rename the file to a name the represents the device and test it in the simulator by pressing the "Add" button in the skins menu.

Submitting the Skin

Assuming your skin file doesn't violate any copyrights we'd appreciate if you submit it to our official skin repository!
If your skin file requires attribution please include that within the skin image...

Start by navigating to github.com/codenameone/codenameone-skins & fork the project. Drag your skin file into the OTA directory. Create a 112x112 icon png file for the skin and drag it into that directory as well.

Next select the Skins.xml file and edit it. Add your skin in a similar way to the other skin files and commit.

Now click the submit pull request button to submit these changes for inclusion in the main repository.
